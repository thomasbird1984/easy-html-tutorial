Basics

What are html (hyper text markup language), css (cascading stylesheets) and scripting languages? I always like to use the anology of a car, likewise I like to explain a website as a car. If you think about a car it has three different major parts that become the car. For instance you have the frame, styling and then the motor. A website has something similar, html is the frame of your website it gives you the core frame of your websites structure. CSS is the styling of the car, in website terms that means that it will control everything you see visually from font colors, to the type of layout and much more. The engine of a car is like the scripting that most websites employ, scripting does things like dropdown menus, information intake from users, storage of data and file uploads are just a couple examples.

Back 10 yrs ago html used to be very basic with only a limited number of tags supported. Tags are the way you tell the browser what any given piece of content is. No a days though html has a multitude of tags that are supported dependent on the browser. Web browsers like chrome, firefox or safari all implement html tags and styles differently in addition to also lacking support for some features. This can be a headache for a developer because you have to know what tags are supported and can even be used. There are great resources though for finding out what you can use such as "caniuse.com"

HTML

Html tags are the building blocks of the web, they're typically formatted like so: <tag_name>Content to be contained</tag_name>. Html tags almost always have a starting and ending tag, in rare cases with tags like <img /> there are no closing tags instead the end of the tag contains a / signifying the end of that tag.

Many times html can be used to identify the kind of content its holding, for instance a <p> tag contains paragraph content and an <a> anchor tags contain links to other places in a page or another website. Typically html tags always have starting and ending tags at the beginning and end of whatever they contain.

CSS

CSS, simply put is the styling of your website, it controls everything from font sizes, spacing and all visual aspects of the application you're developing.

Major parts of the website:

  • Selectors
  • CSS Rule format

Selectors are how we can identify the html elements that we want to style. There are 3 main ways to identify the tags we are trying to target. First is the most basic, the actual tag name. Second and most important we can identify them by class names or id's. Third we can identify tags by combinations of tag names, classes, ids and attributes.

Here is some example html markup and some example selectors:

<div class="wrap"></div>
<ul></ul>
<p id="test"></p> .wrap {
    width: 600px;
    margin: 0 auto;
}

ul {
    list-style: circle;
}

#test {
    font-family: helvetica, arial, verdana;
}

Rules have 2 major parts, the name of the rule and the actual rule. For instance, background-color: red;. Here you see a `:` this is what divides the name of rule from the actual rule. At the end you have a `;` which denotes that this is the end of the rule. If it is the last rule in the set of rules then it doesn't technically require you to put an ending `;` but I do it because it's safer and so that when you go to add more rules you don't forget to add the `;`.

So now you know about selectors and rules lets put it altogether.

Javascript

Javascript is a scripting language, there are two main types of scripting languages, clientside and server side. Client side scripting language operate on the users actual browser, server side scripting languages operate on the server or host computer. The only clientside scripting language is javascript which has many different versions, frameworks and recently started being used as actual servers which is traditionally reserved for server side languages.

An example of some of the most popular server side scripting languages would be php, python, ruby, java and swift just to name a few. Of those php is the most popular, it powers thousands of websites world wide and is to build popular cms (content management systems) and frameworks like Wordpress, Drupal, Laravel, Symfony and so many more. Java is another popular server side language, most notably known for being the scripting language that powers the android platform. Swift is the programming language used to build native ioS applications, it replaced c++ pronounced c sharp.

Back to javascript, javascript is huge language with so many frameworks built on top of it. It handles complex user interactions like tabular data, dropdown menus, image sliders and much more. It's most popular uses are manipulating html and css, it can dynamically can change styles and markup to reflect new interactions.

Simple Homepage

For this example we are going to build a simple one page example of websites homepage. This will have a simple header area, navigation, content area with a sidebar on the left and a main content area on the right and finally a simple footer with some copyright information

Let's start with the most basic and stripped down html markup:

<!doctype html>
<html>
  <head>
    <title>Simple Example</title>
  </head>
  <body>
  </body>
</html>

For this example we will reference an outside stylesheet with this line of code:

<link href="style.css" rel="stylesheet" />

So your code should look like this now:

<!doctype html>
<html>
  <head>
    <title>Simple Example</title>
    <link href="style.css" rel="stylesheet" />
  </head>
  <body>
  </body>
</html>

For the rest of this example we will only be writing code in the style.css and in between the <body></body> sections. So from here on out whenever we are adding code I will let you know whether it belongs in the styles or body.

So let's first start with a simple wrapper, this markup will go between the body tags. We will add all of the necessary containers to the markup for all of our major areas like our header, navigation, content sidebar and main and our footer

<div class="wrapper">
  <div class="header">
  </div>
  <div class="navigation">
  </div>
  <div class="content-wrapper">
    <div class="content-sidebar">
    </div>
    <div class="content-main">
    </div>
  </div>
  <div class="footer">
  </div>
</div>

From here let's add some very basic content, to essentially identify more easily the visual areas.

<div class="wrapper">
  <div class="header">
    <h2>This is the header area</h2>   </div>
  <div class="navigation">
    <ul>
      <li><a href="">Home</a></li>
      <li><a href="">About</a></li>
      <li><a href="">Blog</a></li>
      <li><a href="">Contact</a></li>
    </ul>
  </div>
  <div class="content-wrapper">
    <div class="content-sidebar">
      <h3>Sidebar</h3>
    </div>
    <div class="content-main">
      <h3>Main Content Area</h3>
    </div>
  </div>
  <div class="footer">
    <h3>Footer area</h3>
  </div>
</div>

Next let's add some selectors for the various pieces of our markup to the styles:

CSS Flex a little deeper

Flex allows you to basically say, if I were to divide the width of the container by 4, then we are able to say we want one div to have 1/4 and another to have 3/4 of the containers space. So if we wanted to make both parts equal width we would set the flex property to .content-sidebar { flex: 2; } and .content-main { flex: 2; }. This could be simplified down to .content-sidebar { flex: 1; } and .content-main { flex: 1; }

Really flex can be divided by any number and then each section or column can be given a fraction of that to determine what size it will take up. So for instance, lets say the container is divided into 100 equal parts and we would have have to elements inside. Let's also say we want one element to be 1/4 and the other element to be 3/4 of the containers width. So that would be 25 and 75, which can be simplified to 1/4 and 3/4. So it would be flex: 1 and flex: 3.

Below is an example of a container divided into 4 equal parts with each contained block taking up 1/4 of the containers space, hence flex = 1.

flex: 1;
flex: 1;
flex: 1;
flex: 1;

Below is an example of a container divided into 4 equal parts with each contained block taking up 1/4 of the containers space and another taking up 3/4. Hence flex = 1 in one block and flex = 3 in the remaining block.

flex: 3;
flex: 1;
